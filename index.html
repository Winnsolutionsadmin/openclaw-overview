🚦 Autonomy Framework

Three-tier decision model — what the system can do without asking

🟢 GREEN — Act Silently

  • Archive promotional emails
  • Internal file operations
  • CRM field updates (touch dates, notes)
  • Install missing npm dependencies
  • Create daily memory files
  • Update error patterns
  • Archive old logs (>7 days)

🟡 YELLOW — Act + Log

  • Disable a broken plugin
  • Restart stalled channel
  • Install/update ClawHub skills
  • Auto-fix known error patterns
  • Update cron payload text
  • Create CRM entries for new contacts
  • Generate reach-out drafts (staged)

🔴 RED — Requires Approval

  • Any outbound communication
  • Schedule or cancel meetings
  • Edit config or credentials
  • Disable/enable cron jobs
  • Delete anything
  • Spend money
  • Anything external-facing

Self-Healing Loop

Every heartbeat: scan logs → classify errors → auto-fix GREEN tier → log YELLOW tier → alert RED tier. System recovers from common failures without human intervention.

Karpathy Loop

Inspired by Andrej Karpathy's AutoResearch. Takes any skill, runs it against binary eval assertions, scores output, makes ONE change per cycle, re-tests, keeps if improved, reverts if not. Loops autonomously until interrupted or perfect score.

loop: eval → score → mutate → re-eval → keep/revert

Safety Mechanisms

  • Full SHA-256 backup before every change
  • Automatic rollback if regression detected
  • 50K token cap per improvement loop
  • All changes logged to Notion System Changes DB
  • Version tracked in CHANGELOG.md
  • Separate evaluator invocation (no self-grading)
